# Agent Core 开发指令规范 (AGENTS.md)

作为本项目的 AI 开发助手，在进行任何代码生成或修改时，必须严格遵守以下工程规范：

## 1. 破旧立新策略 (Zero Backward Compatibility)
- **直接覆盖**：功能更新或重构时，**禁止**编写任何向后兼容代码或迁移脚本。
- **清理历史**：不需要保留旧接口或废弃代码，直接采用最新的架构设计覆盖旧内容。

## 2. 文档结构与收口原则
- **根目录纯粹化**：严禁在根目录的 `README.md` 写入子模块细节。
- **禁止子 README**：严禁在任何子模块目录内创建 `README.md`。
- **集中管理**：所有子模块的详细说明必须统一放置在根目录下的 `docs/` 文件夹中，严格按照 `docs/<module-name>.md` 命名。

## 3. 强制同步更新纪律
- **文档最后更新原则**：任何功能代码修改完成后，流程的最后一步**必须**是检查并更新相关文档。
- **关键变动同步**：一旦修改了子模块的“接口定义”、“环境变量”或“启动方式”，必须强制同步更新对应的 `docs/<module-name>.md`。

## 4. 环境变量标准
- **所见即所得**：环境变量的命名必须以当前最新的代码逻辑为唯一基准。
- **零冗余**：严禁在代码或配置中保留任何废弃的环境变量别名。

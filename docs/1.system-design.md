# 1号文档：系统设计（现行架构）

## 1.1 目标与约束

### 设计目标

1. 以 MCP 为唯一外设接入协议，支持多 Server 动态扩展；
2. 将“有新消息”与“消息正文”解耦：只通知、后拉取；
3. 强制 Tool-as-Action：用户可见回复必须走 `send_*` 工具；
4. 避免通道硬编码，新增渠道尽量不改大脑核心逻辑。

### 强约束

1. 唤醒第一步必须调用 `*__fetch_unread_messages`；
2. 普通文本仅为内心独白，不可直接面向用户；
3. 工具命名统一 `server_alias__tool_name`；
4. 当前单进程内存实现优先，保持结构可迁移到 Redis/DB。

## 1.2 架构组成

### `mcp_servers/cli`

1. 提供消息 API 与 MCP 工具；
2. 持有未读池；
3. 在 `user` 消息写入后广播 `new_message_alert`。

### `aichan/mcp_hub`

1. 连接管理：启动/停止多个 MCP 会话；
2. 工具管理：`list_tools` 聚合并包装为 `StructuredTool`；
3. 通知监听：从 `ServerNotification` 解析唤醒事件；
4. 事件分发：写入 `asyncio.Queue[WakeUpEvent]`。

### `aichan/agent`（`WakeUpAgentRuntime`）

1. 消费 `WakeUpEvent`；
2. 合并同一时段事件批次，减少重复推理；
3. 绑定动态工具，执行 LangGraph reason/tools 循环；
4. 审计是否满足“首步 fetch + 仅工具外显”规则。

### `aichan/main.py`

1. 启动 `MCPManager`；
2. 启动 `WakeUpAgentRuntime`；
3. 提供 `/health`（连接数、工具数、唤醒队列深度）。

## 1.3 关键设计决策

### 决策 A：通知只传“有新消息”

实现：

1. 网关发送 `new_message_alert`；
2. 大脑不信任通知正文，不直接用通知当消息；
3. 正文统一从 `fetch_unread_messages` 拉取。

收益：

1. 通知载荷小，延迟低；
2. 去掉推送大消息的耦合；
3. 多通道时统一模型不变。

### 决策 B：未读池 drain 语义

实现：

1. `fetch_unread_messages` 一次返回全部未读；
2. 返回后原子清空；
3. `ai` 消息不进入未读池，避免自激循环。

收益：

1. 拉取逻辑简单；
2. 易于保证“至少一次感知”；
3. 可与 future ack 机制平滑升级。

### 决策 C：工具统一前缀命名

实现：`server__tool`，例如 `localhost_9000__send_cli_message`。  
收益：解决多服务同名冲突，日志可追溯，路由稳定。

## 1.4 事件与工具协议

### WakeUpEvent 结构（Hub 内部）

字段：

1. `server_name`
2. `event`
3. `channel`
4. `message_id`
5. `received_at`
6. `raw_payload`

### Tool 聚合行为

1. `get_all_tools(refresh=True/False)` 返回 `StructuredTool` 列表；
2. 每个工具调用最终落到 `ClientSession.call_tool`；
3. 调用结果统一走 `parse_call_tool_result` 解析。

## 1.5 运行时行为约束

### 首步约束

运行时会记录“第一条 tool call 是否包含 `fetch_unread_messages`”。  
当前实现是**强提示 + 运行时告警**，不满足时会记 warning（不强行中断流程）。

### 输出约束

1. 若本轮无 `send_*` 调用，记录内心独白并结束；
2. 若有 `send_*` 调用，视为对用户产生可见输出；
3. 运行时不做“自动文本兜底发送”，避免破坏 Tool-as-Action。

## 1.6 可扩展性路径

新增 `qq` 通道最小步骤：

1. 新增 `mcp_servers/qq`，实现 `send_qq_message` + `fetch_unread_messages` + `new_message_alert`；
2. 在 `MCP_SERVER_URLS` 增加该 server URL；
3. 重启大脑，`MCPManager` 自动连接并聚合新工具。

无需修改：

1. `WakeUpAgentRuntime` 主循环；
2. `MCPManager` 事件分发结构；
3. 工具命名冲突策略。

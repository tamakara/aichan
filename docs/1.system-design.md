# 1号文档：AIChan 设计文档

## 1.1 设计目标
AIChan 的核心目标不是“让所有输入尽快触发模型”，而是建立一个可控、可解释、可扩展的认知循环。
系统采用统一感知入口、统一调度中枢、统一推理核心的设计，确保行为稳定且可观测。

## 1.2 全面 Pull（拉取）模式与单核大脑

### 1.2.1 设计定义
- 全面 Pull：外部刺激只能作为信号进入 `Nexus` 队列，不能直接触发 LLM 推理。
- 单核大脑：`Brain` 作为唯一消费者，以单线程语义顺序拉取任务并执行。

### 1.2.2 设计原则
- `Nexus` 负责“何时处理”，`Brain` 负责“如何思考”。
- 输入侧是生产者，推理侧是消费者，边界清晰。
- 任意时刻只允许一个主推理上下文处于执行态。

### 1.2.3 设计收益
- 并发可控：避免多输入并发直冲 LLM 导致上下文竞争。
- 顺序稳定：队列天然提供时序一致性。
- 可扩展：新增输入源只需实现生产者协议并入队。

## 1.3 拟人化注意力机制

### 1.3.1 核心认知假设
AIChan 采用“单焦点注意力”模型：
- 大脑每个时刻只聚焦一个刺激。
- 其他刺激进入等待队列，等待注意力释放。

### 1.3.2 注意力循环
1. 感知：插件采集外界刺激并形成标准信号。
2. 排队：信号进入 `Nexus` 中央队列等待处理。
3. 聚焦：`Brain` 取出一条信号，独占当前注意力。
4. 推理：执行 LLM 推理与工具调用。
5. 释放：完成后归还注意力，进入下一条信号。

### 1.3.3 注意力边界
- `Brain` 是唯一允许访问 LLM 的认知核心。
- 插件不负责推理，只负责感知和动作执行。
- 队列长度反映“待处理注意力负载”，可直接用于系统压测与运行观测。

## 1.4 统一能力观
- `plugins` 统一承载感知与动作能力。
- 同一插件可双工：既可生产信号，也可作为工具被调用。
- `Brain` 看到的是能力集合，而非“输入插件/输出插件”的硬编码分裂。

## 1.5 设计约束
- 禁止任何路径绕过 `Nexus` 直接触发 `Brain` 推理。
- 禁止多个并行推理主循环争用同一会话上下文。
- 所有扩展必须遵守“入队 -> 拉取 -> 推理 -> 动作”主干链路。

# 2号文档：项目结构与代码职责

## 2.1 顶层结构

```text
.
├─ aichan/                 # 大脑进程（MCPHub + WakeUpRuntime）
├─ mcp_servers/            # 通道网关（当前仅 CLI）
├─ docs/                   # 架构与运行文档
├─ cli.py                  # 本地终端客户端
└─ docker-compose.yml
```

```text
aichan/

   ├─ pyproject.toml

```

### `aichan/main.py`

1. 应用生命周期入口；
2. 装配并启动 `MCPManager` 与 `WakeUpAgentRuntime`；
3. 提供健康检查与运行状态输出。

### `aichan/agent/src/agent/wakeup_runtime.py`

1. 唤醒事件消费主循环；
2. LangGraph 推理图（reason -> tools -> reason）；
3. Tool-as-Action 约束审计与日志落点。

### `aichan/mcp_hub/src/mcp_hub/manager.py`

1. MCP 多服务连接管理；
2. 工具发现与 `StructuredTool` 包装；
3. `new_message_alert` 通知解析与 `WakeUpEvent` 入队。

### `aichan/mcp_hub/src/mcp_hub/models.py`

1. `MCPServerConfig`：连接配置；
2. `WakeUpEvent`：统一唤醒事件结构。

### `aichan/mcp_hub/src/mcp_hub/tools_wrapper.py`

1. JSON Schema -> Pydantic 参数模型；
2. MCP 调用返回结果规范化；
3. 统一构建 LangChain `StructuredTool`。

## 2.3 `mcp_servers/cli/`（Gateway）结构

```text
mcp_servers/cli/
├─ app.py
├─ server.py
├─ mcp_server.py
├─ message_store.py
├─ models.py
├─ settings.py
├─ Dockerfile
└─ tests/
```

### `app.py`

1. 组装 FastAPI + MCP 路由；
2. 处理 `POST /v1/messages`；
3. 在 `user` 消息写入后触发 `new_message_alert` 广播。

### `mcp_server.py`

1. MCP 工具声明与调用分发；
2. `send_cli_message` / `fetch_unread_messages` 实现；
3. 会话广播器管理（活跃 session 维护与通知发送）。

### `message_store.py`

1. 内存消息存储；
2. 未读池维护（仅收 `user`）；
3. `fetch_unread_messages` 原子 drain。

### `models.py`

1. `ChatMessage`：通道消息；
2. `UnreadMessage`：拉取未读契约结构；
3. `SendMessageRequest`：网关写入请求模型。

## 2.5 维护规则（结构层）

1. 新通道优先新增 `mcp_servers/<channel>/`，不改 `wakeup_runtime` 主循环；
2. 工具命名冲突统一由 `mcp_hub` 层解决，禁止在业务层手工拼接路由；
3. 文档与目录变更必须同步更新本文件；
4. 删除模块时必须同时删除 `pyproject.toml` 中的 workspace/source 引用。

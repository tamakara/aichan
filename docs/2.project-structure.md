# 2号文档：项目结构与代码职责

## 2.1 顶层结构

```text
.
├─ aichan/                 # 大脑进程（MCPHub + AgentRuntime）
├─ mcp_servers/            # 通道网关（当前仅 CLI）
├─ docs/                   # 架构与运行文档
├─ cli.py                  # 本地终端客户端
└─ docker-compose.yml
```

### `aichan/main.py`

1. 应用生命周期入口；
2. 装配并启动 `MCPManager` 与 `AgentRuntime`；
3. 提供健康检查与运行状态输出。

### `aichan/agent/src/agent/agent_runtime.py`

1. 唤醒事件消费主循环；
2. 编排循环调度、图执行、规则审计组件；
3. 仅保留运行时生命周期接口（start/stop）。

### `aichan/agent/src/agent/runtime_*.py`

1. `runtime_loop.py`：唤醒循环调度与异常隔离；
2. `runtime_cycle.py`：单轮执行与结果汇总；
3. `runtime_graph.py`：LangGraph 推理图构建与执行；
4. `runtime_rules.py`：首步 fetch 硬约束、send 工具识别、内心独白提取；
5. `runtime_context.py`：唤醒上下文消息构建。

### `aichan/mcp_hub/src/mcp_hub/manager.py`

1. MCPHub 对外编排入口；
2. 统一协调连接层、唤醒层、工具目录层、工具执行层；
3. 对外暴露 async 管理接口。

### `aichan/mcp_hub/src/mcp_hub/{connections,session,wakeup,tool_*}.py`

1. `session.py`：MCP custom notification 兼容与 wakeup handler 绑定；
2. `connections.py`：多服务连接生命周期与失败回收；
3. `wakeup.py`：`WakeupSignal` 生成与全局 event 状态；
4. `tool_catalog.py`：工具发现、路由快照、`StructuredTool` 包装；
5. `tool_executor.py`：原始 MCP 工具调用与结果解析。

### `aichan/mcp_hub/src/mcp_hub/models.py`

1. `MCPServerConfig`：连接配置；
2. `WakeupSignal`：统一唤醒信号结构。

### `aichan/mcp_hub/src/mcp_hub/tools_wrapper.py`

1. JSON Schema -> Pydantic 参数模型；
2. MCP 调用返回结果规范化；
3. 统一构建 LangChain `StructuredTool`。

## 2.3 `mcp_servers/cli/`（Gateway）结构

```text
mcp_servers/cli/
├─ app.py
├─ server.py
├─ mcp_server.py
├─ message_store.py
├─ models.py
├─ settings.py
├─ Dockerfile
└─ tests/
```

### `app.py`

1. 组装 FastAPI + MCP 路由；
2. 处理 `POST /v1/messages`；
3. 在 `user` 消息写入后异步入队 `aichan/wakeup` 广播任务。

### `mcp_server.py`

1. MCP 工具声明与调用分发；
2. `send_message` / `fetch_unread_messages` / `fetch_message_history` 实现；
3. 会话广播器管理（活跃 session 维护与通知发送）。

### `message_store.py`

1. 内存消息存储；
2. 未读池维护（仅收 `user`）；
3. `fetch_unread_messages` 原子 drain。

### `models.py`

1. `ChatMessage`：通道消息；
2. `UnreadMessage`：拉取未读契约结构；
3. `SendMessageRequest`：网关写入请求模型。

## 2.5 维护规则（结构层）

1. 新通道优先新增 `mcp_servers/<channel>/`，不改 `agent_runtime` 主循环；
2. 工具命名冲突统一由 `mcp_hub` 层解决，禁止在业务层手工拼接路由；
3. 文档与目录变更必须同步更新本文件；
4. 删除模块时必须同时删除 `pyproject.toml` 中的 workspace/source 引用。

## 2.2 术语对齐
为与 1 号文档统一，本文术语按下表使用：

| 代码层模块 | 术语别名 | 说明 |
| --- | --- | --- |
| `core` | 基因组 | 全局实体、接口契约、配置与日志 |
| `plugins` | 感官与外设 | 对外界采集与执行能力的统一边界 |
| `nexus` | 中央神经枢纽 | 异步信号缓冲与心跳调度中枢（可扩展） |
| `brain` | 大脑皮层 | LLM 推理与工具编排入口 |
| `memory` | 海马体 | 记忆检索与长期存储扩展层 |

## 2.3 分层职责
- `core`（基因组）：提供配置、日志、接口契约与共享实体。
- `plugins`（感官与外设）：统一承载工具能力与通道能力。
- `nexus`（中央神经枢纽）：当前提供队列与心跳能力，作为未来多入口调度底座。
- `brain`（大脑皮层）：执行 LLM 推理与工具调用流程。
- `memory`（海马体）：承接短期/长期记忆扩展。

## 2.4 核心运行拓扑

```mermaid

    U[User]
    M[main.py]
    C[CLI Loop(cli.py)]

    B[Brain]

    L[LLM]
    N[Nexus Queue(Reserved)]

    U --> C

    A --> B
    B --> L
    B --> T
    A -- append reply --> C
    N -. future scheduling .-> A
```

## 2.5 主链路
1. `main.py` 完成插件注册与 `Agent` 组装，并启动 `cli.py` 的交互循环。
2. 用户输入被写入通道消息（`role=user`）。
3. CLI 循环发送 `AgentSignal(channel=...)` 给 `Agent.process_signal(...)`。
4. `Agent` 按通道游标拉取增量消息并调用 `Brain` 推理。
5. `Brain` 按需调用已绑定的工具插件并生成最终文本。
6. `Agent` 将最终文本写回通道（`role=assistant`），CLI 刷新显示。

## 2.6 关键模块落点
- `main.py`
  - 组装 `plugins` 与 `brain`，构建 `Agent`。
  - 启动 CLI 事件循环。
- `cli.py`
  - 承载交互式 CLI 输入/输出循环。
  - 负责消息写入、信号发送与增量刷新显示。
- `packages/nexus/src/nexus/agent.py`
  - 执行信号处理、增量拉取、推理调用与回写。
- `packages/nexus/src/nexus/hub.py`
  - 提供队列与心跳能力，作为未来 Pull 调度扩展点。
- `packages/plugins/src/plugins/registry.py`
  - 统一注册插件并汇总可绑定给 LLM 的工具插件。
- `packages/plugins/src/plugins/base.py`
  - 定义 `BasePlugin`、`BaseToolPlugin`、`BaseChannelPlugin`。
- `packages/brain/src/brain/brain.py`
  - 维护 LangGraph 推理图与工具调用闭环。

## 2.7 运行约束
- `Brain` 是唯一 LLM 调用入口。
- CLI 只负责交互与显示，不直接调用 `Brain`。
- 通道能力由编排层调用，工具能力由 LLM 调用。
- 当前为单进程信号驱动链路；多入口并发场景优先演进到 `Nexus` 统一调度。

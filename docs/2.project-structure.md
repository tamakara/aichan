## 2.2 术语对齐
为与 1 号文档统一，本文术语按下表使用：

| 代码层模块 | 术语别名 | 说明 |
| --- | --- | --- |
| `core` | 基因组 | 全局实体、接口契约、配置与日志 |
| `plugins` | 感官与外设 | 对外界采集与执行能力的统一边界 |
| `nexus` | 中央神经枢纽 | 异步信号缓冲与心跳调度中枢（可扩展） |
| `brain` | 大脑皮层 | LLM 推理与工具编排入口 |
| `memory` | 海马体 | 记忆检索与长期存储扩展层 |

## 2.3 分层职责
- `core`（基因组）：提供配置、日志、接口契约与共享实体。
- `plugins`（感官与外设）：统一承载工具能力与通道能力。
- `nexus`（中央神经枢纽）：当前提供队列与心跳能力，作为未来多入口调度底座。
- `brain`（大脑皮层）：执行 LLM 推理与工具调用流程。
- `memory`（海马体）：承接短期/长期记忆扩展。

## 2.4 核心运行拓扑

```mermaid

    U["User"]
    M["main.py"]
    C["CLI Loop(cli.py)"]
    N["NexusHub Queue + Heartbeat"]

    B["Brain"]
    T["Tool Plugins"]
    L["LLM"]

    U --> C

    M --> N

    N --> A
    A --> B
    B --> L
    B --> T

```

## 2.5 主链路
1. `main.py` 完成插件注册与 `Agent` 组装，将 `Agent` 绑定到 `NexusHub` 并启动 heartbeat。
2. 用户输入被写入通道消息（`role=user`）。
3. CLI 循环发送 `AgentSignal(channel=...)` 到 `NexusHub.push_signal(...)`。
4. `NexusHub` 后台心跳线程串行消费信号并调用 `Agent.process_signal(...)`。
5. `Agent` 按通道游标拉取增量消息并调用 `Brain` 推理。
6. `Brain` 按需调用已绑定的工具插件并生成最终文本。
7. `Agent` 将最终文本写回通道（`role=assistant`），CLI 增量刷新显示。

## 2.6 关键模块落点
- `main.py`
  - 组装 `plugins` 与 `brain`，构建 `Agent`。
  - 绑定并启动 `NexusHub` 心跳，最后启动 CLI 事件循环。
- `cli.py`
  - 承载交互式 CLI 输入/输出循环。
  - 负责消息写入、向 `NexusHub` 发信号、增量刷新显示。
- `packages/nexus/src/nexus/agent.py`
  - 执行信号处理、增量拉取、推理调用与回写。
- `packages/nexus/src/nexus/hub.py`
  - 提供统一队列与心跳消费，负责将信号调度到 `Agent`。
- `packages/plugins/src/plugins/registry.py`
  - 统一注册插件并汇总可绑定给 LLM 的工具插件。
- `packages/plugins/src/plugins/base.py`
  - 定义 `BasePlugin`、`ToolPlugin`、`ChannelPlugin`。
- `packages/brain/src/brain/brain.py`
  - 维护 LangGraph 推理图与工具调用闭环。

## 2.7 运行约束
- `Brain` 是唯一 LLM 调用入口。
- CLI 只负责交互与显示，不直接调用 `Brain` 或 `Agent`。
- 通道能力由编排层调用，工具能力由 LLM 调用。
- 当前为单进程 + 后台 heartbeat 信号驱动链路；多入口并发场景统一进入 `Nexus` 调度。

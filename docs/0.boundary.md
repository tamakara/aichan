# 0号文档：系统边界（Pull + Tool-as-Action）

## 0.1 文档目标

本文件定义 AICHAN 当前版本的**职责边界、状态归属、对外契约与非目标**。  
读完本文件后，应能回答：

1. 谁负责唤醒、谁负责取消息、谁负责回复；
2. 哪些状态放在网关，哪些状态放在大脑；
3. 哪些行为是被禁止的（例如模型直接输出用户可见文本）。

## 0.2 组件边界总览

### AICHAN Brain（`aichan/`）

负责：

1. 管理 MCP Server 连接生命周期（建立、断开、重连容错）；
2. 聚合 MCP 工具并提供给推理运行时；
3. 监听网关的唤醒通知（`new_message_alert`）；
4. 在唤醒后驱动 Agent 执行 `fetch_unread_messages -> 推理 -> send_*` 工具动作闭环；
5. 记录内心独白、工具调用和运行时日志。

不负责：

1. 网关消息落库与消息 ID 生成；
2. 终端 UI 展示细节；
3. 通道协议业务细节（QQ/CLI 的具体接入逻辑由各自 MCP Server 实现）。

### MCP Gateway / MCP Server（`mcp_servers/`）

负责：

1. 提供 MCP 工具（例如 `send_cli_message`、`fetch_unread_messages`）；
2. 持有通道消息与未读池状态（未读消息的唯一来源）；
3. 在收到外部 `user` 消息时广播 `new_message_alert` 通知；
4. 对外提供消息 API（`/v1/messages`、`/v1/events`）供 UI 使用。

不负责：

1. 执行大模型推理；
2. 决策“是否回复/如何回复”；
3. 维护大脑内部事件队列。

### Terminal Client（`cli.py`）

负责：

1. 人类输入采集与展示；
2. 通过网关 API 发送与读取消息；
3. 展示来自 AI 的回复。

不负责：

1. MCP 协议处理；
2. 工具调用路由；
3. 推理状态管理。

## 0.3 核心状态归属

### 网关侧状态（权威状态）

1. 全量消息列表（含自增 `id`）；
2. 未读消息池（按 `channel` 分类）；
3. 活跃 MCP 会话集合（用于广播唤醒通知）。

### 大脑侧状态（运行时状态）

1. MCP 连接与会话对象；
2. 工具路由表（`server__tool`）；
3. `WakeUpEvent` 队列；
4. 当前批次推理临时上下文。

### 约束

1. 未读消息以网关为准；大脑不自行缓存“未读真相”；
2. 用户可见输出必须来自 `send_*` 工具副作用，不允许由模型纯文本直出；
3. 大脑重启后，不依赖本地持久化恢复未读；恢复由网关 `fetch_unread_messages` 保证。

## 0.4 通信契约边界

### 唤醒契约（Gateway -> Brain）

通知类型：`notifications/progress`  
判定字段：`progressToken == "new_message_alert"`  
消息体：`message` 是 JSON 字符串，最小结构：

```json
{
  "event": "new_message_alert",
  "channel": "cli",
  "message_id": 12
}
```

### 拉取契约（Brain -> Gateway）

工具：`fetch_unread_messages()`  
语义：返回当前全部未读并**原子清空**（drain）。  
返回项最小字段：

```json
{
  "channel": "cli",
  "message_id": 12,
  "sender": "user",
  "text": "你好",
  "created_at": "2026-04-04T06:00:00+00:00"
}
```

### 回复契约（Brain -> Gateway）

工具：`send_{channel}_message`（当前已实现 `send_cli_message`）  
语义：写入通道消息，供 UI 消费；`ai` 消息默认不回流到未读池。

## 0.5 非目标（当前版本明确不做）

1. 不兼容旧轮询触发链路（已删除）；
2. 不实现跨会话强一致事务；
3. 不实现多租户隔离；
4. 不实现消息重放审计平台；
5. 不在大脑侧保存长期对话历史数据库。

# 0号文档：文档边界与职责说明

## 0.1 目的
本文件定义 AIChan 文档体系的边界，确保“理念、架构、实现”分层表达，避免重复和职责混杂。

## 0.2 文档编号与职责

### 0号文档（本文件）
- 角色：文档总览与边界约束。
- 回答问题：每份文档应该写什么、不应该写什么。
- 不写内容：具体架构细节、模块调用流程、实现代码。

### 1号文档（设计文档）
- 文件：`docs/1.system-design.md`
- 角色：系统设计理念与原则（Why）。
- 回答问题：为什么采用当前设计、核心认知模型是什么。
- 不写内容：目录树、文件清单、部署步骤、历史迁移说明。

### 2号文档（架构文档）
- 文件：`docs/2.project-structure.md`
- 角色：当前系统结构与运行拓扑（What/How）。
- 回答问题：模块如何分工、如何协作、关键路径在哪里。
- 不写内容：设计动机辩论、历史兼容策略。

## 0.3 写作约束
- 各文档自洽，不依赖“跳转到别处才能读懂”。
- 不写迁移过程、兼容策略、历史包袱。
- 术语保持一致：`plugins`、`nexus`、`brain`、`memory`、`core`。

## 0.4 维护规则
- 新增文档前，先在本文件登记编号与边界。
- 编号稳定，不复用，不随版本重排。
- 若内容跨边界，优先拆分而非混写。
